# 오델로 게임 기획서 (GDD)

## 1. 게임 개요

웹 브라우저에서 플레이할 수 있는 8x8 오델로 게임을 구현한다.
한 화면에서 사용자가 AI와 대전하는 인간 대 AI 모드를 제공한다.
AI는 흑돌, 사용자는 백돌을 사용하며 AI의 게임 플레이 레벨을 선택할 수 있다.

## 2. 게임 범위

### 2.1 포함 사항

- 인간 대 AI 게임
- 흑돌 AI와 백돌 사용자
- 초급, 중급, 고급 AI 난이도
- 오델로의 기본 착수 및 돌 뒤집기 규칙
- 현재 차례와 돌 개수 표시
- 유효한 착수 위치 표시
- 자동 패스, 게임 종료 및 결과 표시
- 새 게임 시작

### 2.2 제외 사항

- 외부 ChatGPT 또는 Codex API 연동
- 로컬 인간 대 인간 게임
- 온라인 멀티플레이
- 로그인, 전적 및 랭킹
- 제한 시간
- 게임 저장 및 불러오기

## 3. 게임 규칙

### 3.1 보드와 돌

- 보드는 8열 x 8행, 총 64칸으로 구성한다.
- 두 플레이어는 각각 흑돌과 백돌을 사용한다.
- 게임 시작 시 중앙 네 칸에 돌을 다음과 같이 배치한다.
  - 4행 4열: 백돌
  - 4행 5열: 흑돌
  - 5행 4열: 흑돌
  - 5행 5열: 백돌
- 흑돌 플레이어가 먼저 시작한다.
- 흑돌은 AI가 플레이하고 백돌은 사용자가 플레이한다.

### 3.2 AI 난이도

- 초급 AI는 유효한 착수 위치 중 하나를 무작위로 선택한다.
- 중급 AI는 모서리, 가장자리, 상대의 다음 유효 수와 즉시 뒤집는 돌 수를 평가해 한 수를 선택한다.
- 고급 AI는 여러 수 앞을 탐색하고 모서리, 위치 안정성, 기동성 및 게임 종료 점수를 평가해 한 수를 선택한다.
- 모든 난이도의 AI는 오델로 규칙상 유효한 위치에만 착수한다.
- 같은 평가의 후보가 여러 개이면 안정적인 좌표 순서로 하나를 선택할 수 있다.
- AI 계산은 브라우저 안에서 수행하며 외부 API 키나 네트워크 연결을 요구하지 않는다.

### 3.3 유효한 착수

- 돌은 비어 있는 칸에만 놓을 수 있다.
- 새로 놓은 돌과 같은 색의 기존 돌 사이에 상대 돌이 한 개 이상 연속으로 끼어 있어야 한다.
- 가로, 세로, 대각선을 포함한 8방향 중 한 방향 이상에서 상대 돌을 뒤집을 수 있어야 유효한 착수로 인정한다.
- 유효하지 않은 칸을 선택하면 보드 상태와 현재 차례는 변경되지 않는다.

### 3.4 돌 뒤집기

- 돌을 놓은 뒤 해당 위치를 기준으로 8방향을 검사한다.
- 새 돌과 기존 자기 돌 사이에 연속해서 놓인 상대 돌을 모두 자기 색으로 뒤집는다.
- 여러 방향에서 조건을 만족하면 해당하는 모든 방향의 돌을 한 번에 뒤집는다.
- 돌 뒤집기가 완료된 후 상대 플레이어의 턴으로 진행한다.

### 3.5 패스

- 현재 플레이어에게 유효한 착수 위치가 없으면 해당 턴은 자동으로 패스한다.
- 패스 사실을 화면에 안내하고 상대 플레이어의 턴으로 변경한다.
- 상대 플레이어에게 유효한 착수 위치가 있으면 게임을 계속한다.
- 두 플레이어 모두 연속해서 착수할 수 없으면 게임을 종료한다.

### 3.6 게임 종료와 승패

- 다음 조건 중 하나를 만족하면 게임을 종료한다.
  - 보드의 모든 칸이 돌로 채워진 경우
  - 두 플레이어 모두 연속해서 유효한 수를 둘 수 없는 경우
- 게임 종료 시 보드에 놓인 흑돌과 백돌의 개수를 비교한다.
- 돌이 더 많은 플레이어가 승리한다.
- 돌 개수가 같으면 무승부로 처리한다.
- 게임이 종료된 뒤에는 새 게임을 시작하기 전까지 추가로 돌을 놓을 수 없다.

## 4. 화면 및 상호작용

화면은 다음 정보를 제공해야 한다.

- 8x8 게임 보드
- 흑돌과 백돌의 현재 개수
- 현재 차례인 플레이어
- 현재 플레이어가 돌을 놓을 수 있는 모든 유효한 위치
- 자동 패스 안내
- 게임 종료 시 승자 또는 무승부 결과
- 새 게임 버튼
- 초급, 중급, 고급 난이도 선택
- AI가 생각 중임을 알리는 상태

사용자가 유효한 위치를 선택하면 돌 배치, 돌 뒤집기, 돌 개수 갱신 및 턴 변경이 즉시 화면에 반영되어야 한다.
AI 차례에는 보드 입력을 비활성화하고 AI가 선택한 착수를 자동으로 화면에 반영해야 한다.
새 게임 버튼을 선택하면 현재 난이도를 유지하면서 보드, 현재 차례, 돌 개수, 패스 및 게임 결과를 모두 초기 상태로 되돌린다.
난이도를 변경하면 선택한 난이도로 새 게임을 즉시 시작한다.

## 5. 상태 및 구조 요구사항

게임 상태는 최소한 다음 값을 관리해야 한다.

- 64칸의 보드 상태: 빈칸, 흑돌 또는 백돌
- 현재 플레이어
- 흑돌과 백돌의 개수
- 현재 플레이어의 유효한 착수 위치
- 연속 패스 여부 또는 횟수
- 게임 진행 중 또는 종료 상태
- 게임 결과
- AI 난이도

게임 규칙 계산은 화면 코드와 분리한다.
착수 가능 여부 검사, 뒤집을 돌 계산, 착수 적용, 패스 및 종료 판정은 UI에 의존하지 않는 게임 로직으로 구현한다.
AI의 착수 선택은 UI와 분리된 순수 JavaScript 함수로 구현한다.
AI 전략은 난이도별로 교체할 수 있어야 하며 게임 reducer는 인간과 AI의 착수를 같은 상태 전이로 처리한다.

## 6. 수용 기준

- 새 게임을 시작하면 표준 중앙 배치가 표시되고 흑돌의 차례가 된다.
- 시작 시 흑돌과 백돌의 개수는 각각 2개이다.
- 현재 플레이어가 둘 수 있는 위치만 유효한 위치로 표시된다.
- 유효하지 않은 위치를 선택해도 보드와 차례가 바뀌지 않는다.
- 유효한 위치에 돌을 놓으면 8방향에서 포획된 모든 상대 돌이 뒤집힌다.
- 착수 후 돌 개수와 현재 차례가 정확하게 갱신된다.
- 현재 플레이어가 둘 곳이 없으면 자동으로 패스되고 안내가 표시된다.
- 양쪽 플레이어 모두 둘 곳이 없거나 보드가 가득 차면 게임이 종료된다.
- 게임 종료 시 돌 개수에 따라 흑 승리, 백 승리 또는 무승부가 표시된다.
- 게임 종료 후 보드를 선택해도 상태가 변경되지 않는다.
- 새 게임을 선택하면 모든 게임 상태가 초기화된다.
- 새 게임을 시작하면 AI 흑돌이 먼저 자동으로 착수한다.
- AI 차례에는 사용자가 보드에 착수할 수 없다.
- 사용자는 백돌 차례에 표시된 유효한 위치에 착수할 수 있다.
- 초급, 중급, 고급 중 난이도를 선택할 수 있다.
- 난이도를 변경하면 선택한 난이도를 적용한 새 게임이 시작된다.
- 각 난이도의 AI는 항상 현재 보드의 유효한 수를 선택한다.

## 7. 테스트 요구사항

게임 규칙 로직에 대해 최소한 다음 항목을 검증한다.

- 초기 보드 배치와 흑 선공
- 가로, 세로 및 대각선 방향의 유효한 착수 판정
- 한 번의 착수로 여러 방향의 돌을 동시에 뒤집는 경우
- 상대 돌 뒤에 자기 돌이 없는 경우 뒤집히지 않는 동작
- 빈칸이 아닌 위치와 뒤집을 돌이 없는 위치의 착수 거부
- 한 플레이어만 착수할 수 없을 때의 자동 패스
- 양쪽 모두 착수할 수 없을 때의 게임 종료
- 보드가 가득 찼을 때의 게임 종료
- 흑 승리, 백 승리 및 무승부 판정
- 새 게임 실행 후 상태 초기화
- 초급 AI의 유효한 수 선택
- 중급 AI의 모서리 및 상대 기동성 우선 평가
- 고급 AI의 탐색 결과와 게임 종료 점수 반영
- AI 턴의 자동 착수와 사용자 입력 차단
- 난이도 변경 후 새 게임 및 선택값 유지
